你是一个全栈开发专家。请帮我构建一个基于 **FastAPI + DeepSeek API + 纯原生前端** 的本地刷题小工具。

### 项目目标
创建一个运行在本地的“期末速通”工具。
1. **后端**：使用 Python FastAPI 作为中间层，负责请求 DeepSeek API 并清洗数据，解决跨域问题。
2. **前端**：一个单文件 HTML，负责输入题目文本、请求本地后端、并渲染刷题界面。

### 技术栈
- **后端**：Python 3.x, FastAPI, Uvicorn, Httpx, Python-dotenv。
- **前端**：HTML5, 原生 JavaScript, Tailwind CSS (CDN引入)。
- **AI模型**：DeepSeek-V3 (`https://api.deepseek.com/chat/completions`)。

### 请生成以下 4 个文件的完整代码：

#### 1. `requirements.txt`
包含项目所需的 Python 依赖库：
- fastapi
- uvicorn
- httpx
- python-dotenv
- pydantic

#### 2. `.env`
- 包含环境变量 `DEEPSEEK_API_KEY`，值为 `sk-replace-with-your-key`。

#### 3. `main.py` (后端核心)
- **初始化**：加载 `.env`，初始化 FastAPI。
- **CORS配置 (关键)**：允许所有来源 (`allow_origins=["*"]`)，确保本地 HTML 文件直接打开可以访问。
- **数据模型**：定义 Pydantic 模型接收前端的 `{ "text": "..." }`。
- **核心接口** `POST /convert`：
  1. 接收文本。
  2. 使用 `httpx` (异步) 请求 DeepSeek API。
     - **System Prompt**：
       "你是一个数据清洗专家。将输入的题目文本转换为 JSON Lines 格式。
       要求：
       1. 每行输出一个合法的 JSON 数组。
       2. 数组格式严格为：[\"题干\", \"答案\", [\"选项A\", \"选项B\", \"选项C\", \"选项D\"]]。
       3. 答案只保留选项字母（如 'A'）。
       4. 不要输出 Markdown 标记，不要输出解释。"
  3. **数据清洗逻辑**：
     - 获取 AI 返回的 content。
     - 去除可能存在的 Markdown 符号（如 ```json）。
     - 按行 split，对每一行进行 `json.loads` 解析，忽略解析失败的空行。
  4. 返回格式：`{ "status": "success", "data": [ ...解析后的数组... ] }`。

#### 4. `index.html` (前端界面)
- **UI 风格**：使用 Tailwind CSS，现代简洁，左右分栏或上下布局。
- **功能区 1：输入**
  - 大文本框 (Textarea) 输入题目。
  - “开始转换”按钮。点击后按钮变灰并显示“处理中...”。
- **功能区 2：刷题卡片**
  - 默认隐藏，获取数据后显示。
  - 遍历后端返回的数组渲染题目。
  - **交互**：
    - 显示题干。
    - 列出 4 个选项（使用 `div` 或 `button`）。
    - 点击选项：如果选项索引与答案匹配（A=0, B=1... 或直接比对字母），显示绿色背景；否则显示红色。
    - 增加一个“显示答案”的小按钮。
- **逻辑**：
  - 使用 `fetch` 请求 `http://127.0.0.1:8000/convert`。
  - 处理 loading 状态和错误提示 (alert)。
